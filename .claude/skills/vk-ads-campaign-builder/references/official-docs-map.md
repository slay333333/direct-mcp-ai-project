# Official VK Docs Map

Используй этот файл как карту, а не как замену документации. Разделы ниже — официальная справка VK Рекламы (VK Help, `help/general`) и API-справочник (VK API, `doc/api`) на `ads.vk.com`; в скобках указан путь раздела. Конкретные поля и ограничения перед live-write сверяй на сайте (или через WebFetch) и в `docs/CLAUDE-VK.md`.

## Обязательные источники

- Репозиторный справочник MCP: `docs/CLAUDE-VK.md`.
- Покрытие инструментов: `docs/VK-ADS-TOOLS.md`.
- Официальная справка VK Рекламы — VK Help (`help/general`).
- Официальная API-документация — VK API (`doc/api`). Поля объектов и ресурсов смотри здесь или в схемах MCP-инструментов.

## Создание и редактирование

- VK Help `start/creating` — структура кампания → группа → объявление, цели, бюджеты, таргетинги, URL-параметры, универсальные объявления.
- VK Help `start/budget_limits` — бюджет обязателен, минимум 300 руб. для дневного и общего лимита, для Дзена 10 000 руб.
- VK Help `start/ad_limits` — новые аккаунты до 300 объявлений, дневной лимит создания 200 объявлений.
- VK Help `start/campaign_status` — черновик, модерация, активна, остановлена, лимит, баланс, завершена, удалена, отклонена.
- VK Help `start/edit_campaign` — редактирование кампаний, групп, объявлений; массовая правка в кабинете до 50 объектов.

## Сайты, пиксель, события

- VK Help `sites/sites` — запуск рекламы сайта, требования к лендингу, выбор кликов или событий пикселя, рекомендация бюджета на 5-10 событий в день.
- VK Help `sites/pixel` — создание/установка пикселя, события, автонайденные события, Data Layer, доступы.
- VK Help `sites/visual_event` — визуальный конструктор событий, до 4 отслеживаемых элементов в одном событии, события нельзя удалить, можно менять категорию/название/ценность.
- VK Help `sites/site_attribution` — PostClick, PostView, total; PostView для событий пикселя, выбранных целевым действием, окно 7 дней.
- VK Help `sites/audience_tags` — аудиторные теги для сбора аудиторий, не для оптимизации как целевое действие.

## Лид-формы

- VK Help `lead_forms/lead_forms` — типы форм, поля, вопросы, результат, юридические данные, уведомления, интеграции, оценка лидов.
- VK Help `lead_forms/lead_ads_tips` — меньше полей и вопросов, быстрый оффер, бонус/скидка, скорость обработки заявок.

## Ecommerce, каталоги, динамический ретаргетинг

- VK Help `ecomm/feed_campaign` — реклама товаров сайта/приложения, типы "привлечение" и "динамический ретаргетинг", связь каталога с источником событий.
- VK Help `ecomm/feed_catalog/ecomm_catalog` — создание, редактирование, доступы, источники событий, обновление каталога.
- VK Help `ecomm/feed_catalog/ecomm_catalog_requirements` — фид по ссылке без редиректов, файл до 8 ГБ по ссылке или 100 МБ upload, до 4 млн товаров, UTF-8/Windows-1251, без HTML-тегов.
- VK Help `ecomm/dyn_ret/ecomm_web_dynrem` — обязательные события: просмотр карточки товара, корзина/add_to_cart, покупка; `product_id` должен совпадать с фидом, `value` передаёт стоимость.

## Mini Apps, Senler, сообщества

- VK Help `network/mini_app` — объект "Вконтакте, ОК, Mini Apps, Дзен" → "Mini Apps и игры"; результат равен выбранному целевому действию.
- VK Help `network/vkwebapptrackevent` — `VKWebAppTrackEvent`, типовые события `subscribe`, `lead`, `purchase`, `add_to_cart`, `registration`, `login` и др.
- VK Help `community/engagement` — цели сообщества: подписка, сообщение, продвижение поста; требования к постам и креативам.
- В `docs/CLAUDE-VK.md` для Mini Apps source of truth: `Package.priced_event_type = 43`, `priced_goal.source_id = tracker.id`, `priced_goal.name = event.name`.

## Агентства

- VK Help `agency/agency_clients` — клиенты агентства, списания с клиента, требования подтверждённых реквизитов для присоединения существующих клиентов.
- В `docs/CLAUDE-VK.md` агентский режим работает через `client_id`; агентские инструменты сами без `client_id`.
